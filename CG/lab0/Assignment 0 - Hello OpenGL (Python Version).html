1. Introduction

For 3D programming assignments this semester, we will be using OpenGL and GLSL  as the platform for rendering scenes. The first task is to ensure that you have OpenGL set up for your particular development environment.

A simple and basic OpenGL program is supplied with the assignment. Your task is to compile and run the application on whatever programming system that you plan on using to do your development for the remainder of the course If all is well, the application should produce a window with a special message.

Important note: We strongly suggest that you spend the time to complete it on whatever system you will be using for future OpenGL assignments,  Completing this assignment will help you ensure that you will be able to compile and execute a working OpenGL program; taking the time to do this now means that you won't have to spend time debugging your OpenGL installation on later assignments!

2. Programming Environment

For all lab assignments this semester, we will be providing you with a programming framework to which you will add your solution. This framework will be slightly different for each assignment, so you should be sure to use the correct version for the assignment you are completing. This document describes the Python version

For this assignment, the framework includes the following:

  • ShaderSetup - a support module that reads, compiles, and links GLSL shader programs.

  • shader.vert, shader.frag - a simple GLSL vertex shader.

  • alt.vert, alt.frag - a simple GLSL fragment shader.

  • helloOpenGL - the main function for the application.

This program will accept both keyboard input (characters '1', '2', and 'q') and mouse left-button clicks. Experiment with them to see how the program reacts; this behavior is similar to what the frameworks for many of the later assignments will do.

The driver program (helloOpenGL) will determine whether or not your OpenGL implementation supports OpenGL 3.2 and GLSL 1.50. (Note: some of the things you'll be doing in your shader programs later in the course will require GLSL 1.50, so this "workaround" may not be usable in those assignments!).  Please contact the instructor ASAP if the shaders do not compile on your system.

6. Notes

6.1. Installing Libraries

If you are doing this (and subsequent) assignments on your own system, you will need to be sure you have a recent OpenGL version, as some of the later work in the course will require at least OpenGL 3.2 and GLSL 1.50. For Python, we will be using GLUT for windowing and user input which is distributed with the Python version of OpenGL.

The Python bindings for OpenGL is called PyOpenGL. The instructions for downloading and installing this package can be found at http://pyopengl.sourceforge.net/.

The assignments also make use of the numpy package which should already be installed on your machine for the 2D assignments.

6.2. Shader Files

All of the programming assignments for this course make use of OpenGL shader files. These files have .vert and .frag extensions.

The assignments are set up assuming that these files are in the same folder as the executable.
